## [Unreleased]

### 🔐 Безопасность
- Добавлено **прозрачное шифрование чувствительных данных** конфигурации
  (cookies Playerok, токен Telegram, реквизиты для вывода средств).
  Значения хранятся на диске зашифрованными (`enc::…`), а код продолжает работать
  с обычными строками. Ключ шифрования — в `bot_settings/.secret_key` или в
  переменной окружения `PAP_SECRET_KEY`. Legacy-конфиги в открытом виде
  продолжают работать и мигрируют автоматически при следующем сохранении.
  Шифрование привязано к пути основного конфига, поэтому конфиги модулей не
  затрагиваются. Пароль доступа к Telegram-боту намеренно остаётся в открытом
  виде — для возможности его восстановления.

### ✨ Добавлено
- 💾 **Бэкап и восстановление** (`backup.py`): архивирование `bot_settings/`
  (вместе с ключом шифрования) и `bot_data/` в локальную папку `backups/` с
  защитой от zip-slip при восстановлении. Команды: `create`, `list`, `restore`.
  Интегрировано в Linux-CLI (`plpap backup` / `plpap restore`), Windows-меню и
  в меню самого Telegram-бота (раздел **💾 Бэкап**: создать / отправить в чат / список).
- 🧪 Набор автотестов (`pytest`) для валидаторов, логики конфигов, шифрования и бэкапа.
- 🧹 Линтер **ruff** с конфигурацией под кодовую базу и хуки **pre-commit**.
- 🤖 GitHub Actions **CI** (линтинг + тесты на каждый push и pull request).
- 🐳 **Docker** и **docker-compose** для запуска в контейнере.
- 📓 Этот файл истории изменений.

### 🐛 Исправлено
- **Пагинация банковских карт** авто-вывода (`_load_withdrawal_info`): цикл
  оставлял только последнюю страницу карт — выбранная карта с ранней страницы не
  находилась. Теперь карты собираются со всех страниц.
- **Импорт модулей из архива** (`states_settings.py`): детект `has_init` содержал
  двойной слэш `//__init__.py` (не срабатывал) и падал с `StopIteration` на
  архивах с файлами в корне. Заменён на корректную проверку `"__init__.py" in names`.
- **`download_update`** (`updater.py`): добавлен таймаут (60 c) — раньше загрузка
  обновления могла зависнуть навсегда.
- **`check_new_releases_task`**: блокирующий сетевой запрос вынесен из event-loop
  (`run_in_executor`) — фоновая проверка обновлений больше не подвешивает бота.
- **`return` внутри `finally`** в `get_json` (`settings.py`, `data.py`) — паттерн
  проглатывал любые исключения (включая `KeyboardInterrupt`). Заменён на
  безопасное чтение с best-effort записью дефолта.
- **`send_message`** (`plbot/playerokbot.py`): конструкция `for/else` всегда
  перезатирала реальный текст ошибки generic-строкой — теперь в лог попадает
  настоящее исключение.
- Изменяемые значения по умолчанию у аргументов-списков/словарей заменены на
  `None`-сентинелы (`call_bot_event`, `call_playerok_event`, `run_async_in_thread`,
  `run_forever_in_thread`, `send_message`).
- `getattr(obj, "const")` заменены на прямой доступ к атрибутам; `raise ... from e`
  в обработчике инициализации модуля.

### 🛠️ Изменено
- Заменены все «голые» `except:` на `except Exception:` (59 мест) — Ctrl+C и
  системные сигналы больше не проглатываются.
- В карточках товара и сделки теперь отображается эмодзи-индикатор статуса.
- Точечная чистка кода: неиспользуемые переменные и импорты, дублирующаяся
  функция, сравнения с `None` через `is`.
